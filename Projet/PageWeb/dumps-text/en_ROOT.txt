   ROOT is an [102]object-oriented [103]computer program and [104]library
   developed by [105]CERN. It was originally designed for [106]particle
   physics [107]data analysis and contains several features specific to
   the field, but it is also used in other applications such as
   [108]astronomy and [109]data mining. The latest minor release is 6.32,
   as of 2024-05-26.^[110][3]

Description

   [[111]edit]

   CERN maintained the [112]CERN Program Library written in [113]FORTRAN
   for many years. Its development and maintenance were discontinued in
   2003 in favour of ROOT, which is written in the [114]C++ programming
   language. ROOT development was initiated by René Brun and Fons
   Rademakers in 1994. Some parts are published under the [115]GNU Lesser
   General Public License (LGPL) and others are based on [116]GNU General
   Public License (GPL) software, and are thus also published under the
   terms of the GPL. It provides [117]platform independent access to a
   computer's graphics subsystem and [118]operating system using
   [119]abstract layers. Parts of the abstract platform are: a
   [120]graphical user interface and a [121]GUI builder, container
   classes, [122]reflection, a C++ [123]script and [124]command line
   [125]interpreter ([126]CINT in version 5, [127]cling in version 6),
   object [128]serialization and [129]persistence.

   The packages provided by ROOT include those for
     * [130]Histogramming and [131]graphing to view and analyze
       [132]distributions and [133]functions,
     * [134]curve fitting (regression analysis) and minimization of
       [135]functionals,
     * [136]statistics tools used for [137]data analysis,
     * [138]matrix algebra,
     * [139]four-vector computations, as used in [140]high energy physics,
     * standard [141]mathematical functions,
     * [142]multivariate data analysis, e.g. using [143]neural networks,
     * image manipulation, used, for instance, to analyze
       [144]astronomical pictures,
     * access to distributed data (in the context of the [145]Grid),
     * [146]distributed computing, to parallelize [147]data analyses,
     * [148]persistence and [149]serialization of objects, which can cope
       with changes in class definitions of persistent data,
     * access to [150]databases,
     * [151]3D [152]visualizations (geometry),
     * creating files in various graphics formats, like [153]PDF,
       [154]PostScript, [155]PNG, [156]SVG, [157]LaTeX, etc.
     * interfacing [158]Python code in both directions,
     * interfacing [159]Monte Carlo [160]event generators.

   [161][220px-Atlas_ROOT_plot.png] The ATLAS experiments presented on 4
   July 2012 the status of the Standard Model Higgs search. All the plots
   presented that day were created in ROOT.

   A key feature of ROOT is a data container called tree, with its
   substructures branches and leaves. A tree can be seen as a sliding
   window to the raw data, as stored in a file. Data from the next entry
   in the file can be retrieved by advancing the index in the tree. This
   avoids memory allocation problems associated with object creation, and
   allows the tree to act as a lightweight container while handling
   buffering invisibly.

   ROOT is designed for high [162]computing efficiency, as it is required
   to process data from the [163]Large Hadron Collider's experiments
   estimated at several [164]petabytes per year. As of 2009^[165][update]
   ROOT is mainly used in [166]data analysis and [167]data acquisition in
   [168]particle physics (high energy physics) experiments, and most
   current^[169][update] experimental plots and results in those subfields
   are obtained using ROOT.

   The inclusion of a C++ interpreter ([170]CINT until version 5.34, Cling
   from version 6.00) makes this package very versatile as it can be used
   in interactive, scripted and compiled modes in a manner similar to
   commercial products like [171]MATLAB.

   On July 4, 2012 the ATLAS and CMS LHC's experiments presented the
   status of the Standard Model Higgs search. [172]All data plotting
   presented that day used ROOT.

Criticisms

   [[225]edit]

   Criticisms of ROOT include its difficulty for beginners, as well as
   various aspects of its design and implementation. Frequent causes of
   frustration include extreme [226]code bloat, heavy use of global
   variables,^[227][4] and an overcomplicated class hierarchy. From time
   to time these issues are discussed on the ROOT users mailing
   list.^[228][5]^[229][6] While scientists dissatisfied with ROOT have in
   the past managed to work around its flaws,^[230][7] some of the
   shortcomings are regularly addressed by the ROOT team. The CINT
   interpreter, for example, has been replaced by the Cling
   interpreter,^[231][8] and numerous bugs are fixed with every release.

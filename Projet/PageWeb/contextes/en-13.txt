Root directory

   [ ] 21 languages
--
   From Wikipedia, the free encyclopedia
   First or top-most directory in a hierarchy
   For other uses, see [78]Root (disambiguation).
   [79][220px-OpenIndiana_2015_Console_LS.png] View of the root directory
   in the [80]OpenIndiana operating system

   In a [81]computer [82]file system, and primarily used in the [83]Unix
   and [84]Unix-like [85]operating systems, the root directory is the
   first or top-most [86]directory in a hierarchy.^[87][1] It can be
   likened to the trunk of a [88]tree, as the starting point where all
   branches originate from. The root file system is the file system
   contained on the same [89]disk partition on which the root directory is
   located; it is the filesystem on top of which all other file systems
   are [90]mounted as the system boots up.^[91][2]
--

   Unix abstracts the nature of this tree hierarchy entirely and in Unix
   and Unix-like systems the root directory is denoted by the / (slash)
   sign. Though the root directory is conventionally referred to as /, the
   directory entry itself has no name - its path is the "empty" part
   before the initial directory separator character (/). All file system
   entries, including mounted file systems are "branches" of this
   root.^[93][1]

chroot

   [[94]edit]

   In UNIX-like operating systems, each [95]process has its own idea of
   what the root directory is. For most processes this is the same as the
   system's actual root directory, but it can be changed by calling the
   [96]chroot [97]system call. This is typically done to create a secluded
   environment to run software that requires legacy libraries and
   sometimes to simplify software installation and debugging. Chroot is
   not meant to be used for enhanced security as the processes inside can
   break out.^[98][3]

Super-root

   [[99]edit]
   See also: [100]Newcastle Connection

   Some Unix systems support a directory below the root directory.
   Normally, "/.." points back to the same [101]inode as "/", however,
   under [102]MUNIX [[103]de], this can be changed to point to a
   super-root directory, where remote trees can be
   mounted.^[104][4]^[105][5] If, for example, two workstations "pcs2a"
   and "pcs2b" were connected via "connectnodes" and "uunite" startup
   script, "/../pcs2b" could be used to access the root directory of
   "pcs2b" from "pcs2a".

--
   Under [107]DOS, [108]OS/2, and [109]Microsoft Windows, each partition
   has a drive letter assignment (e.g. the C partition is labeled C:\) and
   there is no public root directory on it.

Related uses
--

   On many [111]Unixes, there is also a directory named
   /root (pronounced "slash root").^[112][6] This is the [113]home
   directory of the 'root' [114]superuser. On many [115]Mac and [116]iOS
   systems this superuser home directory is /var/root.^[117][6]

   A [118]home page's [119]URL usually points to the root of the
   respective [120]website's [121]domain name.

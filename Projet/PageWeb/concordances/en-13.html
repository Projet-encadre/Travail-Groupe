In a [81]computer [82]file system, and primarily used in the [83]Unixroot
and [84]Unix-like [85]operating systems, the root directory is the
first or top-most [86]directory in a hierarchy.^[87][1] It can beroot
likened to the trunk of a [88]tree, as the starting point where allroot
branches originate from. The root file system is the file system
contained on the same [89]disk partition on which the root directory is
located; it is the filesystem on top of which all other file systemsroot
are [90]mounted as the system boots up.^[91][2]root
--root
root
Unix abstracts the nature of this tree hierarchy entirely and in Unixroot
and Unix-like systems the root directory is denoted by the / (slash)
sign. Though the root directory is conventionally referred to as /, the
directory entry itself has no name - its path is the "empty" partroot
before the initial directory separator character (/). All file systemroot
entries, including mounted file systems are "branches" of thisroot
root.^[93][1]

[[94]edit]root
root
In UNIX-like operating systems, each [95]process has its own idea ofroot
what the root directory is. For most processes this is the same as the
system's actual root directory, but it can be changed by calling the
[96]chroot [97]system call. This is typically done to create a secluded
environment to run software that requires legacy libraries androot
sometimes to simplify software installation and debugging. Chroot is
not meant to be used for enhanced security as the processes inside canroot
break out.^[98][3]root
